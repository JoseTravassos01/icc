Trabalho sobre a importância do github e suas aplicações

O GitHub é uma plataforma essencial para o desenvolvimento de software. Ele oferece controle de versão, facilita a colaboração entre desenvolvedores, compartilha código-fonte aberto, permite a criação de portfólios e oferece integração com várias ferramentas. Além disso, o GitHub possui recursos para gerenciamento de problemas e solicitações de recursos. Em resumo, o GitHub é importante por sua capacidade de promover a colaboração, o compartilhamento e o controle eficiente de código.

Comandos gits e suas funcionalidades

Comando "git init"

O comando git init cria um novo repositório Git vazio em um diretório, configurando o controle de versão do Git para o projeto. Com o repositório inicializado, é possível rastrear alterações, criar branches, fazer commits e controlar o histórico do código-fonte

  1. Criar um repositório em um novo ou reautiliza um que já existe

Comando "git clone"

O comando git clone cria uma cópia local de um repositório Git remoto. Essa cópia permite que você trabalhe no projeto e sincronizandoas alterações com o repositório remoto através de comandos como git push e git pull.

  1. clona um repositório em um novo

Comando "git branch"

O comando git branch no Git permite criar, listar e alterar branches em um repositório. Essa funcionalidade é de extrema importância para gerenciar diferentes trabalhos e controlar as alterações em um projeto.

  1. Lista todas as branches do repositório
  2. Cria uma nova branch chamada branch-nova
  3. Exclui a branch chamada branch-nova
  4. alterna para a branch chamada branch-nome

Comando "git checkout"

A funcionalidade do comando git checkout no Git é alternar entre diferentes branches, visualizar versões específicas do código e manipular o estado do repositório

  1. alterna para a branch chamada branch-nome
  2. Descarta altrações do arquivo que não foram commitadas

Comando "git status"

A funcionalidade do comando git status no Git é fornecer informações sobre o estado atual do repositório

  1. Mostra como está a Working Tree

Comando "git diff"

A funcionalidade do comando git diff no Git é mostrar as diferenças entre diferentes estados do repositório. Ele exibe as alterações feitas nos arquivos entre diferentes commits e branches.

  1. Mostrará as diferenças entre o diretório de trabalho e o index

Comando "git add"

A funcionalidade do comando git add no Git é adicionar arquivos ao índice de staging, preparando-os para serem incluídos no próximo commit.

  1. Adiciona algo para o index

Comando "git commit"

A funcionalidade do comando git commit no Git é confirmar as alterações realizadas nos arquivos adicionados ao índice de staging.

  1. Grava alterações com uma mensagem de commit

Comando "git push"

O comando git push é usado para enviar as alterações confirmadas de um repositório local para um repositório remoto. O git push permite compartilhar o código com outros membros da equipe ou disponibilizá-lo publicamente.

  1. Envia alterações do branch atual para o repositório remoto origin

Comando "git pull"

O comando git pull é usado para atualizar o repositório local com as alterações mais recentes do repositório remoto. O git pull é fundamental para sincronizar o trabalho local com as últimas alterações feitas por outros participantes do projeto.

Comando "git revert"

O comando git revert desfaz alterações específicas em um repositório Git, criando novos commits que revertem as mudanças introduzidas por commits anteriores. Ele preserva o histórico existente, permitindo a correção de erros de maneira segura. O git revert é uma ferramenta para gerenciar o histórico do projeto.

  1. Reverte as alterações pro 2° último commit realizaado

Comando "git merge"

O comando git merge no Git permite combinar as alterações de uma branch com outra. O git merge pode resolver automaticamente as fusões simples ou dar a a opção de fazer manualmente. É uma ferramenta para integrar o trabalho de diferentes branches e manter a consistência do código.

  1. Combina alterações de na atual/desejada

Comando "git stash"

O comando git stash no Git é usado para salvar alterações locais em uma área temporária chamada "stash". Ele permite limpar o diretório de trabalho e a área de staging, deixando-os livres para outras tarefas ou mudanças de branch. Útil para armazenar temporariamente alterações sem perdê-las.

  1. salva alterção local

Comando "git rm"

O comando git rm no Git é usado para remover arquivos do repositório Git e do diretório de trabalho local, git rm pode remover arquivos , diretórios e subdiretórios.

  1. Remove arquivo da Working Tree

Comando "git config"

O comando git config no Git é usado para configurar diversas opções e preferências.O git config é uma ferramenta para personalizar o Git às necessidades individuais e de projeto.

Comando "git reset"

O comando git reset no Git tem a funcionalidade de desfazer alterações.

Comando "git remote"

O comando git remote no Git é usado para gerenciar repositórios remotos. Ele permite visualizar, adicionar e remover repositórios remotos associados a um projeto Git.

  1. Lista os repositórios remotos

Comando "git fetch"

comando git fetch no Git é usado para baixar as alterações mais recentes de um repositório remoto para o repositório local. Ele atualiza as referências locais, permitindo que você sincronize seu repositório local com o estado atual do repositório remoto.

  1. Pega branches do repositório nome-repo

Comando "git show"

O comando git show no Git é usado para exibir informações detalhadas sobre um objeto específico, como um commit, uma tag ou um blob. Ele mostra as alterações, o autor, a data e a mensagem do objeto, permitindo explorar o histórico e os dados relacionados.

  1. Mostra a tag especificada

Comando "git help"

O comando git help no Git é usado para obter informações de ajuda e acessar a documentação relacionada aos comandos do Git. Ele fornece detalhes sobre o uso correto dos comandos, opções disponíveis e conceitos relevantes do Git.

  1. para acessar ajuda quando necessário

Comando "git rebase"

O comando git rebase no Git permite alterar a base de um branch movendo seus commits para outro ponto de partida. Ele reescreve o histórico de commits, aplicando as alterações feitas em um branch em cima de outro.

  1. Aplica alterações de outra-branch na atual

Comando "git blame"

Ele fornece informações sobre o autor, data e commit associados a cada linha específica, permitindo entender quem modificou o código, quando e qual commit introduziu determinadas alterações

  1. Para alterar a base de um branch

Comando "git tag"

O comando git tag no Git é usado para criar, listar e gerenciar tags. As tags são usadas para marcar pontos específicos no histórico do projeto, como versões de lançamento.

  1. Cria a tag com o nome nometag. Para remover a tag, apenas coloca-se o -d no lugar de -s

Explicação de como criar uma branch, de como fazer e aplicar alterações:

Criando uma Branch:

  1. Clique na aba "code" do seu repositório nome_do_repo.
  2. Acima da lista de aquivos, clique no menu suspenso que diz "main"
  3. No campo de texto digite o nome da branch.
  4. Clique em "Creat Branch:read name-edits from main"

Como fazer e aplicar alterações:

  1. Na brancch readme-editsque criamos anteriormente, clique em README.md.
  2. Para começar editar, clique no "lapis".
  3. No campo editor, escreva o texto.
  4. Clique em commit changes....
  5. No campo "propose changes" escreva uma mensagem de commit.
  6. Clique em "propose changes".
